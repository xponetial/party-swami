# Party Swami Multi-Agent Phase Plan

## Recommended Phase Count

Use **6 delivery phases** aligned to your strategy docs, but execute them in **2 waves** so we can parallelize safely.

1. **Phase 0: Foundations + Orchestrator Contract**
2. **Phase 1: Revenue MVP Agents**
3. **Phase 2: Core Experience Agents**
4. **Phase 3: Marketplace Agents**
5. **Phase 4: Growth + Intelligence Agents**
6. **Phase 5: Software Factory Layer**

## Why 6 Phases

- `Agents for Party Swami.rtf` already defines phased rollout (MVP -> core -> marketplace -> growth).
- `Skills for Each Agent.rtf` defines reusable skill architecture that should be established before deep feature expansion.
- `Party Swami Evolution.docx` extends the roadmap to advanced autonomy and software-factory behavior.

This gives us a practical merge of all three documents: monetization first, then defensibility.

## Parallelization Strategy

### Wave A (can run together after Phase 0 baseline)

1. **Track A1: Planning + Orchestration path**
   - Party Planning Agent
   - Orchestrator routing for planning flows
2. **Track A2: Invitation + Content path**
   - Invitation/Card Agent
   - Content/design skills
3. **Track A3: Shopping + Budget path**
   - Shopping Recommendation Agent
   - Budget Agent
4. **Track A4: Execution path**
   - Task/Reminder Agent
   - RSVP/Guest Agent

### Wave B (parallel after Wave A contracts are stable)

1. **Track B1: Vendor Marketplace path**
   - Marketplace Vendor Agent
   - Vendor onboarding capabilities
2. **Track B2: Growth path**
   - Social Media Agent
   - Admin Growth Agent
3. **Track B3: Software Factory path**
   - Internal feature builder workflows
   - Test automation/digital twin scaffolding

## Dependency Rules

1. Phase 0 must complete first (schemas, contracts, guardrails).
2. In Wave A, tracks can run concurrently if they do not alter shared contract files simultaneously.
3. Wave B starts once orchestrator and skill contracts are versioned and frozen for the sprint.
4. Software Factory work should not block product-revenue tracks.

## Expected Outputs Per Phase

1. **Phase 0**
   - Agent contract spec
   - Skill contract spec
   - Shared output schemas
2. **Phase 1**
   - Planning, invitation, shopping production-ready flows
3. **Phase 2**
   - Task/budget/rsvp integrated execution
4. **Phase 3**
   - Vendor matching and onboarding workflows
5. **Phase 4**
   - Growth automation + insights loops
6. **Phase 5**
   - Internal autonomous build/test pipeline v1
